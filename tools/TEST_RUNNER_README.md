# 🧪 测试运行器使用指南

## 📖 简介

测试运行器是一个**可视化测试界面**，让你可以轻松运行和查看测试结果，不需要敲命令行！

## 🚀 快速开始

### 方法 1：双击运行（推荐）

1. 找到项目根目录下的 `run_test_ui.py`
2. 双击运行
3. 界面自动打开！

### 方法 2：命令行运行

```bash
python run_test_ui.py
```

## 🎨 界面说明

### 左侧：测试列表

- **Thread Management 测试** - 线程管理相关的 13 个测试
- **集成测试** - 服务集成测试（4 个）
- **其他功能测试** - 其他功能模块测试（17 个）

每个测试显示：
- 📝 测试名称
- 🔵 状态（未运行/运行中/通过/失败）
- ⏱️ 运行时间

### 右侧：测试结果

显示选中测试的详细信息：
- ✅ 测试状态
- ⏱️ 运行时间
- ❌ 错误信息（如果失败）
- 💡 修复建议

### 顶部：工具栏

- **▶ 运行所有测试** - 运行全部 34 个测试
- **▶ 运行选中测试** - 只运行你选中的测试
- **🗑 清除结果** - 清空所有测试结果

### 底部：状态栏

- 📊 进度条 - 显示测试运行进度
- 📈 统计信息 - 总计/通过/失败/未运行

## 📋 使用步骤

### 1. 运行所有测试

1. 点击 **"▶ 运行所有测试"** 按钮
2. 等待测试完成（会显示进度条）
3. 查看结果统计

### 2. 运行单个测试

1. 在左侧测试列表中**点击**你想运行的测试
2. 点击 **"▶ 运行选中测试"** 按钮
3. 查看右侧的测试结果

### 3. 运行某一类测试

1. 在左侧测试列表中**点击**分类节点（如 "Thread Management 测试"）
2. 点击 **"▶ 运行选中测试"** 按钮
3. 该分类下的所有测试都会运行

### 4. 查看测试详情

1. 点击左侧的任意测试项
2. 右侧会显示该测试的详细信息
3. 如果测试失败，会显示错误信息和修复建议

## ❌ 测试失败了怎么办？

### 查看错误信息

1. 点击失败的测试项
2. 右侧会显示详细的错误信息
3. 查看 **"💡 建议"** 部分

### 常见错误类型

| 错误类型 | 原因 | 解决方法 |
|---------|------|---------|
| `AttributeError` | 代码接口变了 | 需要更新测试脚本 |
| `TypeError` | 参数类型不匹配 | 检查函数调用参数 |
| `AssertionError` | 断言失败 | 检查代码逻辑 |
| `ImportError` | 导入失败 | 检查依赖是否安装 |

### 更新测试脚本

如果界面提示 **"可能需要更新测试脚本"**：

1. 找到对应的测试文件（在 `tests/` 目录下）
2. 查看错误信息，了解哪里出错了
3. 修改测试代码，适配新的接口
4. 重新运行测试

## 💡 使用技巧

### 1. 快速定位问题

- 先运行所有测试，看哪些失败
- 点击失败的测试，查看详细错误
- 根据错误信息修复代码或测试

### 2. 分类运行

- 修改了线程管理代码？只运行 "Thread Management 测试"
- 修改了服务层代码？只运行 "集成测试"
- 添加了新功能？运行 "其他功能测试"

### 3. 持续验证

- 每次修改代码后，运行相关测试
- 确保没有破坏现有功能
- 测试通过后再提交代码

## 🎯 什么时候需要更新测试？

### ✅ 不需要更新（大部分情况）

- 添加新功能（不改旧功能）→ 只需要写新测试
- 修复 bug（不改接口）→ 可能需要新增测试
- 性能优化（不改行为）→ 测试应该还能通过

### ⚠️ 需要更新（少数情况）

- 修改了函数参数 → 需要更新测试
- 修改了函数返回值 → 需要更新测试
- 修改了类的接口 → 需要更新测试
- 删除了功能 → 需要删除对应的测试

## 🌟 示例场景

### 场景 1：日常开发

```
1. 修改了 ThreadManager 的代码
2. 打开测试运行器
3. 选择 "Thread Management 测试"
4. 点击 "运行选中测试"
5. 查看结果 → 全部通过 ✅
6. 放心提交代码！
```

### 场景 2：添加新功能

```
1. 添加了新功能：优先级队列
2. 写了新测试：test_priority_queue.py
3. 打开测试运行器
4. 运行所有测试
5. 新测试通过 ✅，旧测试也通过 ✅
6. 提交代码！
```

### 场景 3：测试失败

```
1. 运行测试 → 5 个失败 ❌
2. 点击失败的测试，查看错误
3. 错误：AttributeError: 'ThreadManager' object has no attribute 'run_task'
4. 原因：修改了接口，run_task 改名为 run_in_thread
5. 更新测试：把 run_task 改为 run_in_thread
6. 重新运行 → 全部通过 ✅
```

## 📞 遇到问题？

如果测试运行器本身有问题：

1. 检查 pytest 是否安装：`pip install pytest`
2. 检查 PyQt6 是否安装：`pip install PyQt6`
3. 查看控制台错误信息
4. 联系开发者
